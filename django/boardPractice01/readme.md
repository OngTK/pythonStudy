### 3. DB 모델링(핵심)

1. User 확장 전략 결정
    - Django 기본 User 그대로 사용 vs CustomUser
2. 게시판 모델 설계
    - Category(이슈/정치/취미/자유)

### 7. “본문 중간 이미지 첨부” 구현(중요 요구사항)

실습 난이도가 올라가는 구간이라 **2단계로 학습**을 추천합니다.

#### 7-1) 1단계(쉽고 확실): 대표 이미지/갤러리 방식

- 글 작성 시 이미지 여러 장 업로드 → 상세에서 본문 아래 “이미지 영역”으로 출력
- 빠르게 요구사항의 “이미지 첨부”를 충족

#### 7-2) 2단계(진짜 본문 삽입): 에디터/업로드 API 방식(추천)

- Summernote / CKEditor / TinyMCE 중 하나 선택
- 이미지 업로드 엔드포인트 구현
- 업로드된 이미지 URL을 본문(content)에 삽입
- 정리 포인트
    - MEDIA 설정, 보안(로그인 사용자만 업로드), 파일 확장자 제한

### 11. 공통 UX/화면 구성(필수 화면 + 추천 화면)
